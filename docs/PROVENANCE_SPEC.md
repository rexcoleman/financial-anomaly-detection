# PROVENANCE SPECIFICATION

<!-- version: 1.0 -->
<!-- created: 2026-03-16 -->

## 1) Purpose

Defines the provenance chain for all experiment outputs in the financial anomaly detection project.

## 2) Provenance Directory

Location: `outputs/provenance/`

| File | Contents | When Written |
|------|----------|-------------|
| `versions.txt` | Python + all key library versions + platform | Start of any experiment run |
| `git_info.txt` | Git commit SHA and dirty state | Start of any experiment run |
| `config_resolved.yaml` | Global config snapshot | Start of reproduction sequence |

## 3) Environment Capture

`versions.txt` schema:

```
python: 3.x.x
numpy: x.x.x
pandas: x.x.x
scikit-learn: x.x.x
xgboost: x.x.x
lightgbm: x.x.x
shap: x.x.x
matplotlib: x.x.x
platform: linux-x86_64
```

## 4) Per-Run Provenance

Each run's `config_resolved.yaml` contains:
- `run_id`: deterministic ID per ARTIFACT_MANIFEST_SPEC
- `git_sha`: commit hash at run time
- `seed`: random seed used
- `timestamp_utc`: ISO 8601 run start time
- All model hyperparameters

## 5) Integrity Chain

```
git_info.txt        → code version
versions.txt        → environment version
config_resolved.yaml → parameter version
```

Together these three files are sufficient to reproduce any result.

## 6) Synthetic Data Provenance

PaySim data is generated by `scripts/generate_synthetic_data.py` with seed 42. The generator script itself is version-controlled, so `git_info.txt` captures the data generation logic.

## 7) Change Control

Changes to provenance file format or requirements need a `CONTRACT_CHANGE` commit.
